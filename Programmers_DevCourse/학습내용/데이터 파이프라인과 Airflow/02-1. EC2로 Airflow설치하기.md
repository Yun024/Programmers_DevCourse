### 리눅스 이해하기
- 우분투 (ubuntu): 리눅스 타입 중의 하나. 다른 타입은 데비안, 레드햇, 페도라 
- ssh: 리눅스 혹은 유닉스 서버에 로그인해주는 프로그램 (터미널)
    + private key와 public key를 사용
- sudo: 보통 슈퍼유저로서 프로그램을 구동할 수 있도록 하는 프로그램이다.
- apt-get: 우분투/데비안 계열의 리눅스에서 프로그램 설치/삭제를 관리해주는 프로그램
    + apt-get update, apt-get install
- su: substitue user의 약자
    +현재 사용 중인 사용자 계정을 로그아웃하지 않고 다른 사용자의 권한을 얻을 때 사용한다
- vi: 텍스트 에디터

## Airflow 코드 프로그램 실행
### Airflow 코드의 기본 구조
- DAG 대표하는 객체를 먼저 만듦
    + DAG 이름, 실행주기, 실행날짜, 오너 등등
- 다음으로 DAG를 구성하는 태스크들을 만듦
    + 태스크별로 적합한 오퍼레이터를 선택
    + 태스크 ID를 부여하고 해야할 작업의 세부사항 지정
- 최종적으로 태스크들간의 실행 순서를 결정

1. 모든 태스크에 공통으로 적용될 설정 작성
```python
default_args = {
   'owner': 'yeojun',
   'email': ['yyj024@naver.com'],
   'retries': 1,
   'retry_delay': timedelta(minutes=3),
}
```
    - `owner` 소유자, `email` 이메일, `retries` 실패 시 재시도 횟수 `retry_delay` 재시도 간격
    - `on_failure_callback` 실패 시 호출 함수 지정 `on_success_callback` 성공 시 호출 함수 지정


2. DAG 객체 작성
```python
from airflow import DAG
dag = DAG(
   "dag_v1",              # DAG name 
   start_date=datetime(2020,8,7,hour=0,minute=00),
   schedule="0 * * * *",  # 매시 0분에 시작, @hourly, @daily, @weekly ...
   tags=['example'],      # 기본으로 example
   catchup=False,         # default=True
   default_args=default_args 
)
```
- catchup default: True
    + 만약 start_date이 실행 기준으로 과거라면, 과거에 실행되지 않은 DAG를 catchup함
    + False로 지정하면 catchup하지 않음
    + full refresh를 하는 DAG라면 False로 지정
    + Incremental update를 하는 DAG의 경우 True로 지정

3. Bash Operator을 사용한 예제
```python
t1 = BashOperator(
   task_id='print_date',
   bash_command='date',
   dag=test_dag)

t2 = BashOperator(
   task_id='sleep',
   bash_command='sleep 5',
   dag=test_dag)

t3 = BashOperator(
   task_id='ls',
   bash_command='ls /tmp',
   dag=test_dag)

t1 >> [ t2, t3 ]
```
- 3개의 태스크로 구성
- t1은 현재 시간 출력
- t2는 5초간 대기 후 종료
- t3는 서버의 /tmp 디렉토리의 내용 출력
- t1이 끝나고 t2와 t3를 병렬로 실행
