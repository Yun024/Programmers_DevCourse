# 데이터 카탈로그
## 데이터 카탈로그 소개
많은 회사들이 데이터 카탈로그를 메인 데이터 거버넌스 툴로 사용하고 있다.
### 데이터 카탈로그
- 데이터 자산 메타 정보 중앙 저장소
- 데이터 거버넌스의 첫 걸음
    + 많은 회사에서 데이터 카탈로그를 데이터 거버넌스 툴로 사용
    + 혹은 데이터 카탈로그위에 커스팀 기능을 구현해서 사용
- 데이터 카탈로그의 중요한 기능
    + (반)자동화된 메타 데이터 수집
    + 데이터 보안 - 보통 메타 데이터만 읽어옴
        - 실제 데이터의 노출로 인한 위험 요소를 감소시킴
- 데이터 자산의 효율적인 관리 프레임워크
    + 다양한 관점에서 데이터를 조직적으로 고나리
    + 비즈니스/데이터 용어 vs 태그
    + 데이터 오너(Business & Technical)
    + 표준화된 문서 템플릿

### 데이터 자산의 종류
- 테이블(DB), 대시보드, ML피쳐, 데이터 파이프라인
+ 문서/메시지(슬랙, JIRA, Github), 사용자(HR시스템)

### 데이터 카탈로그 주요 기능
- 주요 데이터 플랫폼 지원
- 비즈니스 용어집(Business Glossary)
- 주석/문서/태그 등 협업 기능
- 데이터 리니지
- 데이터 모니터링, 감사, 트레이싱
- 강력한 검색 기능(통합 검색, NLP 검색)
- 데이터 추천 기능
- 데이터 유저 퍼소나(마케팅 분석가)

### 주요 데이터 플랫폼 지원
- DW & Data Lake
    + Redshift, Snowflake, BigQuery
- BI Tools
    + Looker, Tableau, Redash, PowerBI, Mode, Superset
- ELT 
    + DBT, Spark, Hive, PrestoDB
- ETL Orchestration 
    + Airflow
- NoSQL and others
    + Cassandra, Druid, Elastic Search, Kafka Schema Registry, CSV
- Users
    + Azure AD, LDAP

### DBT는 Template화된 SQL
- DBT코드 = SQL + Jinja template = Template화된 SQL
    + 개발자가 작성한 DBT코드를 타겟 시스템에 맞는 SQL로 바꾸어 실행해줌
- 모델부터 작성하는데 이는 하나의 SELECT 문이라고 보면됨
    + A model == A SELECT statement
- 다양한 검증 밥법 추가할 수 있음
    + Generic tests
    + One-off tests
- 테이블의 스냅샷 추가도 가능함

### 비즈니스 용어집(Business Glossary)
- 권한이 있는 사람만 용어 정의 가능
- 계층구조로 관리할 수 있다면 더욱 유용
    + DataHub의 경우 terms와 terms group 존재
- 비즈니스 용어와 Entity 연결
    + 나중에 다른 Entity등과 연결이 가능함
- 태그 vs 비즈니스 용어
    + 태그는 좀더 비공식적인 데이터 분류 방법
    + 비즈니스 용어는 계층 구조 형태의 분류체계를 따라감

### 데이터 리니지
- Dataset-to-dataset
    + 보통 SQL 파싱으로 일어남
- Pipeline
    + 입력 데이터셋 -> Data Pipeline -> 출력 데이터셋
    + Airflow에 lineage backend라는 것이 존재
- Dashboard-to-chart
    + 하나의 차트가 여러 대시보드에 소속가능하기에 필요한 리니지
- Chart-to-dataset
- Job-to-dataflow
    + DBT에 특별한 리니지

### 데이터 거버넌스 관점에서 데이터 카탈로그의 중요성
- 우리가 갖고 있는 데이터 자산에 대한 통합 뷰를 제공
- 생산성 증대: 설문이나 데이터 티켓의 감소로 확인
- 위험 감소: 잘못된 결정과 개인정보등의 전파 방지
- 인프라 비용 감소: 불필요한 정보의 생성 방지와 안 쓰이는 데이터셋 삭제
- 데이터 티켓 감소
- 데이터 변경으로 인한 이슈 감소
    + 컬럼 레벨 리니지와 CI/CD 프로세스 연동

### 데이터 카탈로그 이후 다음 스텝
- 자동화된 데이터 거버넌스 관련 워크플로우를 추가
- 품질 관련 경보 시스템 구현
    + 중요 메타데이터 변경이나 데이터 품질 이슈 발생시 경보
    + 사용자가 관심있는 데이터 자산의 오너 변경시 경보(매출 정의 변경)
- 데이터 관련 지표 리뷰 미팅 운영

### 데이터 카탈로그 트렌드

### 데이터 카탈로그 툴
- 상용제품
    + Alation, Collibra, Atlan, Select Star, Great Expectations
- 오픈소스
    + Amundsen(Lyft), DataHub(LinkedIn), AcrylData(DataHub를 상용화)
- 클라우드 서비스
    + AWS Glue Data Catalog, Google Cloud Data Catalog, 
    + Microsoft Azure Data Catalog(Purview Data Catalog로 통합)
- 자체 툴
    + DataBook(Uber), DataPortal(Airbnb)

### Atlan
- 2018년 창업 스타트업(싱가폴, 인도 기반)
- 슬로건: Modern Data Workspace(3rd generation data catalog)
    + 협업 관련 여러 서비스들과의 연동에 초점(Slack, Jira, Github 등등)
    + Slack에서 "Contextual discussion"이란 것을 지원
    + 데이터 관련 이슈를 바로 Jira와 연동하여 리포트 가능
- 기타 특징
    + 오픈 API를 통한 새로운 메타데이터의 추가 연동이 쉬움(플러그인 마켓플레이스)
    + 프로그래밍이 가능한 봇을 사용해 태스크 자동화 가능

### DataHub
- 2013년에 LinkedIn 내부 프로젝트로 시작했다가 2018년에 오픈소스가 됨
- Acryl Data라는 스타트업으로 2020년 창업
- 슬로건: "A Metadata Platform for the Modern Data Stack" 
- 굉장히 많은 기능이 존재하나 엔지니어 없이는 운영 불가
- DataHub의 장점
    + 메타데이터를 여러 관점에서 볼 수 있게 해줌
    + 데이터 처리와 품질 관리를 더 앞 단계에서 수행하여 비용과 리스크를 줄일 수 있음
    + 메타 데이터를 모니터링하여 변화에 맞게 행동을 취할 수 있음

### Microsoft Azure Purview Data Catalog
- 2022년 4월 Azure Data Catalog가 Purview의 서비스로 리브랜딩
- Purview Data Map이라는 메타데이터 관리 서비스 위에서 동작
    + 메타데이터를 수집하고 리니지 정보 추출과 PII등의 데이터 분류를 담당
- 기타 특징
    + 아주 다양한 데이터 플랫폼을 지원
    + 비즈니스 용어와 태깅과 관련된 기능이 다른 제품보다 잘 되어있음
    + 데이터 거버넌스 제품의 일부 서비스이기에 거버넌스로 확장 용이
