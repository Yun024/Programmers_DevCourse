 ## Airflow Docker 환경 설정
 ### Dokcer기반 Airflow실행
 - docker-compose.yaml 수정
    + _PIP_ADDITIONAL_REQUIREMENTS 수정
    + data 폴더를 호스트 폴더에서 만들고 볼륨으로 공유: 임시 데이터를 저장할 폴더
        - 이를 Docker volume으로 지정해서 나중에 디버깅에 사용
```bash
environment:
    AIRFLOW_VAR_DATA_DIR: /opt/airflow/data  # 환경변수 DATA_DIR: /opt/airflow/data경로 
    # web UI에서는 보이지 않지만 프로그램에서 사용가능 
        # docker exec -it {container ID} airflow variables get DATA_DIR
    _PIP_ADDITIONAL_REQUIREMENTS: 
        ${_PIP_ADDITIONAL_REQUIREMENTS:- yfinance pandas numpy oauth2client gspread} 
        # :- _PIP_ADDITIONAL_REQUIREMENTS 여부에 대한 조건문

volumes:
    ...
    - ${AIRFLOW_PROJ_DIR:-.}/data:/opt/airflow/data

airflow-init:
    ...
    mkdir -p /sources/logs /sources/dags/ sources/plugins /sources/data
    chown -R "${AIRFLOW_UID}:0" /sources/{logs,dags,plugins,data}
```

### .airflowignore
- Airflow의 DAG 스캔 패턴은?
    + airflow.cfg에 설정된 dags_folder가 가리키는 폴더를 서브폴더까지 모두 스캔
        - DAG모듈이 포함된 모든 파이썬 스크립트 실행
        - 새로운 DAG를 찾게 되며 이는 가끔 사고로 이어짐
    + Airflow가 의도적으로 무시해야 하는 DAG_FOLDER의 디렉토리 또는 파일을 지정
    + .airflowignore의 각 줄은 정규식 패턴으로 지정하며 매칭되는 파일들은 무시됨
- project_a ; tenant_[\d]: 아래 파일들을 무시
    + project_a_dag_1.py, TESTING_project_a.py, tenant_1.py, project_a/dag_1.py

### NPS Summary 테이블 생성
- NPS란? Net Promoter Score
    + 10점 만점으로 '주변에 추천하겠는가?'라는 질문을 기반으로 고객 만족도를 계산
    + 10,9점 추천하겠다는 고개(promoter)의 비율에서 0-6점의 불평고객(detractor)의 비율은 뺀 값
        - 7,8점은 아예 계산에 들어가지 않음
- 각자 스키마.nps 테이블 혹은 raw_data.nps 테이블 기준으로 일별 nps summary 생성

### 데이터 파이프라인 실패/경고를 슬랙으로 보내는 방법
- 생성된 Webhook URL을 airflow의 "slack_url" variable로 저장
- slack에 에러 메세지를 보내는 별도 모듈로 개발 "slack.py"
- DAG 인스턴스를 만들 떄 에러 콜백으로 지정
```python
from plugins import slack
raise # 에러 처리
default_args = {
    'on_failure_callback': slack.on_failure_callback,
}
```
