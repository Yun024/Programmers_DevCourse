## 구글 서비스 어카운트 생성
- 구글 클라우드 로그인
    + https://console.cloud.google.com/
- 구글 스프레드시트 API 활성화 필요
    + https://console.cloud.google.com/apis/library/sheets.googleapis.com
- 구글 서비스 어카운트 생성(JSON)
    + https://robocorp.com/docs/development-guide/google-sheets/interacting-with-google-sheets
    + https://denisluiz.medium.com/python-with-google-sheets-service-account-step-by-step-8f74c26ed28e
- JSON 파일의 내용을 google_sheet_access_token 이란 이름의 Variable로 등록
- JSON 파일을 보면 이메일 주소가 하나 존재 
    + 이를 읽고 싶은 구글스프레드시트 파일에 공유. 해당 메일은 iam.gserviceaccount.com로 끝남

## 구글 서비스 어카운트 생성 절차
- 시트 API 활성화 
    1. 사이드바 Credentials 선택
    2. CREATE CREDENTIALS 선택
    3. Service account 선택
    4. API Keys화면에서 방금 생성한 서비스 어카운트 선택
    5. Tab 중 KEYS 선택
    6. Private key 생성 화면에서 JSON을 선택하고 해당 파일을 다운로드(자동)
        + 파일의 내용을 Airflow Variable로 등록(google_sheet_access_token)

# API & Airflow 모니터링
## Airflow API 활성화
- airflow.cfg의 api 섹션에서 auth_backend의 값을 변경
    + auth_backend = airflow.api.auth.backend.basic_auth
- docker-compose.yaml environments설정
    + AIRFLOW__API__AUTH_BACKENDS: 
        'airflow.api.auth.backend.basic_auth,airflow.api.auth.backend.session'
- 해당 커맨드로 확인
    + docker exec -it {container_id} airflow config get-value api auth_backend
- API를 노출시키더라도 VPN 뒤에 숨겨둬야함
    + 같은 네트워크 안의 내부 서비스들만 액세스할 수 있도록
- Airflow Web UI에서 새로운 사용자 추가 (API 사용자)
    + Security -> List Users -> "+"
    + 이후 화면에서 새 사용자 정보 추가(monitor:MonitorUser1)

## API 사용 예
- [API 레퍼런스](https://airflow.apache.org/docs/apache-airflow/stable/stable-rest-api-ref.html#section/Overview)
- [특정 DAG를 API로 Trigger하기](https://airflow.apache.org/docs/apache-airflow/stable/stable-rest-api-ref.html#operation/post_dag_run)
    + curl -X POST --user "airflow:airflow" -H 'Content-Type: application/json' -d '{"execution_date":"2025-02-01T00:00:00Z"}' "http://localhost:8080/api/v1/dags/SQL_to_Sheet/dagRuns"
- [모든 DAG 리스트하기](https://airflow.apache.org/docs/apache-airflow/stable/stable-rest-api-ref.html#operation/get_dags)
    + curl -X GET --user "airflow:airflow" http://localhost:8080/api/v1/dags
- [모든 Variable 리스트하기](https://airflow.apache.org/docs/apache-airflow/stable/stable-rest-api-ref.html#tag/Variable)
    + curl -X GET --user "airflow:airflow" http://localhost:8080/api/v1/variables
- [모든 Config 리스트하기](https://airflow.apache.org/docs/apache-airflow/stable/stable-rest-api-ref.html#tag/Config)
    + curl -X GET --user "airflow:airflow" http://localhost:8080/api/v1/config
- Python에서 실행하기
```python
import requests
from requests.auth import HTTPBasicAuth

url = "http://localhost:8080/api/v1/dags"

dags = requests.get(url, auth=HTTPBasicAuth("airflow","airflow))
print(dags.test) # dags.json()
```

### Variables/Connections Import/Export
- DB에 기록이되어서 Web UI에서 보이는 것들만 출력
    + 환경변수에 등록된 것은 출력되지 않음
```bash
airflow variables export variables.json
airflow variables import variables.json

airflow connections export connections.json
airflow connections import connections.json
```

### DAGS 리스트를 API로 읽고 활성화되어 있는 DAG만 찾기
```python
import requests
from requests.auth import HTTPBasicAuth

url ="http://localhost:8080/api/v1/dags"
dags = requests.get(url, auth=HTTPBasicAuth("airflow","airflow"))
for d in dags.json()["dags"]:
    if not d["is_paused"]:
        print(d["dag_id"])
```

### config API는 기본적으로 막혀있는데 접근하기 위한 방법
- 해당 configureation 섹션과 키(airflow.cfg)
    + webserver 섹션
    + expose_config를 True로 설정(문자열)
- docker-compose.yaml에 적용하기
```bash
environment:
    $airflow-common-env
    AIRFLOW_VAR_DATA_DIR: /opt/airflow/data
    AIRFLOW__WEBSERVER__EXPOSE_CONFIG: 'true'
```
