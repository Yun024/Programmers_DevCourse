1. Docker란?
    - 컨테이너 기반의 가상화 기술
    - 애플리케이션과 그 종속성을 패키지화하여 어디서나 실행할 수 있도록 지원
    - 주요 특징: 경량화, 이식성, 효율성, 이미지 기반
2. Docker 용어 정리
    - Docker Image: Docker Container를 실행하기 위한 템플릿
        + 파일 시스템과 실행 환경이 포함된 불변 객체로, 컨테이너를 생성할 때 사용
    - Dockerfile: Docker 이미지를 생성하기 위한 설명서
        + 이미지 생성에 필요한 명령어(FROM, RUN, COPY 등)가 작성된 텍스트 파일
    - Docker Container: Docker 이미지를 실행한 독립 실행 환경
        + 실제로 애플리케이션이 실행되는 인스턴스
    - Docker Hub(hub.docker.com): Docker 이미지를 공유하는 공식 저장소
        + 전 세계 개발자들이 이미지를 업로드하거나 다운로드할 수 있는 플랫폼
    - Docker Compose: 다중 컨테이너 애플리케이션을 정의하고 관리하기 위한 도구
        + YAML 파일을 사용하여 여러 컨테이너를 정의 및 설정
3. Docker 사용시 주의할 점
    - 이미지 최적화
        + 불필요한 파일 및 레이어를 포함하지 않도록 Dockerfile을 작성
        + 멀티스테이지 빌드를 사용하여 이미지 크기 줄이기
    - 보안
        + 공용 이미지를 사용할 때 신뢰할 수 있는 소스인지 확인
        + 민감한 정보(API 키, 비밀번호)를 Docker 이미지에 포함하지 않도록 주의
    - 리소스 관리
        + 컨테이너 실행 시 CPU 및 메모리 리소스를 제한하는 옵션 설정(--memory, --cpu)
        + 중지된 컨테이너와 사용하지 않는 이미지를 정기적으로 관리(docker system prune)
    - 네트워킹
        + 컨테이너 간 통신을 설정할 때 네트워크 드라이버(브릿지, 호스트, 오버레이 등)이해하고 사용
    - 로깅 및 모니터링
        + 로그 파일 크기 관리(--log opt 사용)
        + 모니터링 도구(e.g. , Prometheus, Grafana)와 통합하여 성능 확인
4. Kubernetes(K8s)란?
    - Container Orchestration 플랫폼으로 대규모 컨테이너 배포,관리,확장,네트워킹을 자동화
    - 주요 특징
        + 자동화된 컨테이너 배포 및 확장: 수동 작업 없이 컨테이너를 자동으로 배포하거나 확장 가능
        + 자체 복구(Self-Healing): 문제가 발생한 컨테이너를 자동으로 재시작, 교체, 재배치
        + 로드 밸런싱: 여러 컨테이너 간 트래픽을 분산하여 부하를 관리
        + 스토리지 오케스트레이션: 외부 스토리지 시스템(EBS, NFS 등)과 쉽게 통합
        + 애플리케이션 배포 전략
            - 롤링 업데이트, 블루-그린 배포, 카나리아 배포
    - K8s 구성 요소
        + Pod: 컨테이너의 논리적 그룹(컨테이너가 최소 단위로 실행)
        + Node: 워커 머신(물리적 또는 가상)
        + Cluster: 여러 Node가 구성된 K8s 시스템
        + Controller: 애플리케이션 상태를 관리(e.g. , Deployment, StatefulSet)
        + Service: Pod 간의 통신 및 외부 네트워킹을 처리
※ Docker와 K8s의 관계
- Docker: 컨테이너 생성 및 실행에 초점
- Kubernetes: 여러 컨테이너를 관리 및 오케스트레이션
- Docker로 컨테이너를 패키징하고 Kubernetes로 클러스터 환경에서 관리
