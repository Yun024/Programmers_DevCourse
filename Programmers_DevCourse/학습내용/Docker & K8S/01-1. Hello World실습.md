# Docker 실습
## 간단한 Hello World 프로그램 실습
### 만들려는 프로그램 개요
- Node.js로 구성된 초간단 웹 서비스
    + app.js가 전부
    + 하지만 node 런타임 환경이 필요
- 실행하기 위해 node app.js
    + yoejun hello-world-docker % node app.js Hello Docker!
- 노드 환경이 설정되어 있어야 함
- 직접 설치/실행시 순서(Dockerfile에 기술하면 Docker Image생성 가능)
    + OS선택
    + Node 설치
    + 코드 복사
    + 프로그램 실행(node app.js)

### Dockerfile의 생성
- Docker에게 소프트웨어 설치 명령을 기술
- 먼저 베이스 이미지를 기술(FROM)
- 다음으로 코드 복사
- 마지막으로 코드 실행
```docker
FROM node:alpine ## OS종류 - alpine:경량 리눅스 모델
COPY ./app       ## 코드 복사에 사용
WORKDIR /app     ## Working directory를 지정
CMD node app.js  ## 실행하는 명령 앞에 지정
```

### Dockerfile 사용 가능 기타 키워드
- ARG
    + Docker Image를 만들 때 사용되는 변수 지정, 최종 이미지에는 들어가지 않음
    + 빌드 타임 인수
- ENV
    + 컨테이너가 실행될 때 사용되는 환경변수
    + 최종 이미지에 저장됨
    + 런 타임 변수
- USER
    + 컨테이너를 실행할 떄 사용할 유저 ID
- EXPOSE
    + 서비스 사용 포트번호
- RUN
    + 반드시 실행되어야 하는 명령들이 지정됨(docker build)
    + RUN apt-get update && apt-get install -y curl

### Dockerfile 키워드 CMD vs ENTRYPOINT
- container가 시작할 때 실행되어야 하는 명령어를 지정하는데 사용(docker run)
    + 굉장히 흡사한 기능을 제공하지만 우선 순위가 있음
- 둘다 한 DOCKERFILE에서 여러번 실행되면 각각 마지막 것만 사용됨
- 아래의 경우 docker run 실행시 동일한 결과가 나옴 
```docker
CMD["command1.sh"]
**CMD["command1.sh"]**

ENTRYPOINT["command1.sh"]
**ENTRYPOINT["command2.sh"]**
```

- CMD나 ENTRYPOINT 중 하나만 지정되면 그게 container가 실행될 때 실행
```docker
FROM debian:buster          
COPY ./myproject
RUN apt-get update...
CMD ["./cmd1.sh"]

docker run my-image         ## ./cmd.sh 가 기본으로 실행됨
docker run my-image cmd2.sh ## cmd2.sh가 실행됨(Override)
```

- 둘이 한 DOCKERFILE에서 같이 지정가능함
- 둘이 같이 사용되면 ENTRYPOINT가 기본 명령이 되고 CMD가 인자를 제공함
- ENTRYPOINT는 --entrypoint 옵션을 통해서만 덮어쓰기가 가능
```docker
FROM debian:buster
COPY ./myproject
RUN apt-get update...
ENTRYPOINT ["entrypoint1.sh"]
CMD["param1","param2"]

docker run my-image         # entrypoint.sh param1 param2
docker run my-image cmd2    # entrypoint.sh cmd2
docker run --entrypoint="/cmd3.sh" my-image
```

- 강의 내용에서 추천 - 최대한 CMD를 사용할 것 

Dockerfile 살펴보기
```python
# VERSION 1.10.9
# AUTHOR: Matthieu "Puckel_" Roisil
# DESCRIPTION: Basic Airflow container
# BUILD: docker build --rm -t puckel/docker-airflow .
# SOURCE: https://github.com/puckel/docker-airflow

FROM python:3.7-slim-buster   ## 이미지 이름, 소프트웨어와 버전
LABEL maintainer="Puckel_"

# Never prompt the user for choices on installation/configuration of packages
ENV DEBIAN_FRONTEND noninteractive  ## ENV 최종 이미지가 실행될때도 사용가능한 환경변수
ENV TERM linux

# Airflow
ARG AIRFLOW_VERSION=1.10.9          ## ARG 빌드할 때만 사용하며 이미지에는 안 들어감
ARG AIRFLOW_USER_HOME=/usr/local/airflow
ARG AIRFLOW_DEPS=""
ARG PYTHON_DEPS=""
ENV AIRFLOW_HOME=${AIRFLOW_USER_HOME}

# Define en_US.
ENV LANGUAGE en_US.UTF-8
ENV LANG en_US.UTF-8
ENV LC_ALL en_US.UTF-8
ENV LC_CTYPE en_US.UTF-8
ENV LC_MESSAGES en_US.UTF-8

# Disable noisy "Handling signal" log messages:
# ENV GUNICORN_CMD_ARGS --log-level WARNING

RUN set -ex \
    && buildDeps=' \
        freetds-dev \
        libkrb5-dev \
        libsasl2-dev \
        libssl-dev \
        libffi-dev \
        libpq-dev \
        git \
    ' \
    && apt-get update -yqq \
    && apt-get upgrade -yqq \
    && apt-get install -yqq --no-install-recommends \
        $buildDeps \
        freetds-bin \
        build-essential \
        default-libmysqlclient-dev \
        apt-utils \
        curl \
        rsync \
        netcat \
        locales \
    && sed -i 's/^# en_US.UTF-8 UTF-8$/en_US.UTF-8 UTF-8/g' /etc/locale.gen \
    && locale-gen \
    && update-locale LANG=en_US.UTF-8 LC_ALL=en_US.UTF-8 \
    && useradd -ms /bin/bash -d ${AIRFLOW_USER_HOME} airflow \
    && pip install -U pip setuptools wheel \
    && pip install pytz \
    && pip install pyOpenSSL \
    && pip install ndg-httpsclient \
    && pip install pyasn1 \
    && pip install apache-airflow[crypto,celery,postgres,hive,jdbc,mysql,ssh${AIRFLOW_DEPS:+,}${AIRFLOW_DEPS}]==${AIRFLOW_VERSION} \
    && pip install 'redis==3.2' \
    && if [ -n "${PYTHON_DEPS}" ]; then pip install ${PYTHON_DEPS}; fi \
    && apt-get purge --auto-remove -yqq $buildDeps \
    && apt-get autoremove -yqq --purge \
    && apt-get clean \
    && rm -rf \
        /var/lib/apt/lists/* \
        /tmp/* \
        /var/tmp/* \
        /usr/share/man \
        /usr/share/doc \
        /usr/share/doc-base

COPY script/entrypoint.sh /entrypoint.sh
COPY config/airflow.cfg ${AIRFLOW_USER_HOME}/airflow.cfg  ## 내 코드를 이미자 안에 코드복사

RUN chown -R airflow: ${AIRFLOW_USER_HOME}  ## 빌드할 때 실행되는 명령

EXPOSE 8080 5555 8793        ## 서비스 사용 포트번호

USER airflow
WORKDIR ${AIRFLOW_USER_HOME}  ## 기본 경로 지정 
ENTRYPOINT ["/entrypoint.sh"]  ## entrypoint에 있는 명령 실행
CMD ["webserver"]   ## entrypoint 명령 실행 이후 파라미터로 붙음
## /entrypoint.sh webserver 
```

### Docker Image생성
- Docker Image이름
    + docker image ls 와 docker images와 동일한 결과를 보여줌
    + docker tag 명령의 경우 별칭을 만들어주는 것(이름을 바꾸는 것이 아님)
        - docker tag hello-world-docker yoejun/hello-world-docker
    + Docker image의 실제 ID는 IMAGE ID임
    + Image 이름 자체는 REPOSITORY 이름과 TAG로 구성됨
        - 한번에 쓰는 경우 :을 사이에 두고 같이 씀
        - redis:13 , yeojun/hello-world-docker:latest
    + docker image ls에서 Repository에 해당
        - Docker Hub에서 다운로드받은 것이라면 어카운트ID(네임스페이스)를 포함할 수 있음
    + 공식 이미지들의 경우에는 네임스페이스가 없음
        - ubuntu:18.04, node:alpine
        
- 아래 코드로 생성
```bash
docker build --platform linux/amd64 -t hello-world-docker. 
# 특정 칩 기반 맥의 경우 ARM 기반 아키텍처로 만들어짐
    # --platform옵션을 사용해 linux/amd64로 지정 
# docker build를 실행하면 Dockerfile에서 RUN 명령이 실행됨 
# -t는 태그를 지정하는 것
# hello-world-docker 이름을 지정
# .은 hello-world-docker 이미지가 지금 디렉토리에 있는 dockerfile로 만들어짐을 뜻함
    # 다른 디렉토리에 있는 dockerfile을 사용할 경우 .뒤에 path를 지정

docker image ls
# docker image가 잘 만들어졌는지 확인
# 이미지 이름 뒤에 콜론을 통해 집어넣으면 버전이 됨(아무 문자열이나 가능) - TAG
    # 아무것도 입력하지 않을 경우 latest가 입력됨

docker run hello-world-docker 
# docker run은 Dockerfile에서 CMD명령이 실행됨
```
### Docker tag란?
- Docker Image의 버전이나 변형을 나타내는 문자열
    + default 값은 latest
    + Docker Image의 부가정보를 나타냄
- Docker Image 이름에서 ":" 뒤에 해당
    + ubuntu:18.04 , bitnami/airflow, node:alpine

### docker run VS docker exec
- docker run은 새로 Container를 실행하는 것
- docker exec는 실행된 Container에 작업을 하는 것
    + 그래서 명령에 Container ID가 필요함
- 두 명령 모두 --user root 혹은 -u root를 통해 루트 유저로 연결가능


## Docker Hub 소개
### Docker Registry에 등록하기
- https://hub.docker.com/ 회원 등록
- Create repository 선택
- 이름을 "hello-world-docker"로 지정 , 속성은 Public
- 터미널로 이동하여 아래 명령을 실행
- 현재 이미지의 repo이름을 yoejun024/hello-world-docker로 변경
```bash
docker image ls
docker tag hello-world-docker:latest yeojun024/hello-world-docker:latest # image name 변경
docker image ls
docker login --username=yeojun024     # dockerhub login
docker push yeojun024/hello-world-docker # dockerhub upload
```
