## MySQL
- Docker Engine이 실행된 것 확인하고 terminal 프로그램 실행
- MySQL docker image를 다운로드
- 다운로드받은 이미지로 Docker container 실행
- MySQL root 계정의 패스워드 찾기
- MySQL shell 실행하기
- docker run --name
    + 기억하기 쉬운 이름을 docker ps로 찾은 Container ID 대신 사용 가능
- docker logs
    + Container쪽에서 생성된 stdout, stderr단의 로그를 읽어옴
    + --follow 옵션을 사용하면 로그가 계속적으로 스트리밍이 됨

```bash
docker pull mysql/mysql-server:8.0
docker run --name=mysql_container mysql/mysql-server:8.0 
docker ps
```

```bash
docker logs mysql_container 2>&1 | grep GENERATED
docker exec -it mysql_container mysql -uroot -p

show database # 에러발생
ALTER USER root@localhost IDENTIFIED BY 'yeojun024';

show database
use mysql
show tables;
```


## Hangman
### 웹서비스를 Docker로 구동해보는 전체 과정 익히기
- hangman_web이라는 repo의 main에 코드가 merge될 때마다 다음을 수행
1. 테스트 수행
2. Docker Image 빌드
    + 이를 위해 Dockerfile 부터 만들기
3. Dokcer Image를 Docker Hub로 푸시
- 위 과정을 Github repo에 Github Actions으로 구현

### 프로그램 소개
- https://github.com/learndataeng/hangman_web
- hangman 프로그램을 flask를 사용하여 웹으로 노출
    + 포트번호는 어디든 바인딩 가능하며 실행할 떄 지정
    + flask 관련 모듈 설치가 필요함: requirements.txt
- 실행 방법
    + python3 -m flask run --host==0.0.0.0 --port=4000
        - app.py를 기본으로 사용함 

### Repo 구성 
- app.py
    + flask의 메인 함수
    + 커맨드라인으로 받은 포트에 바인드하고 요청이 들어오기를 기다림
- requirements.txt
    + pip3 install requirements.txt
    + flask 모듈들을 설치
- test.py
    + app.py에 있는 코드의 유닛 테스트 로직이 들어가 있음
    + CI/CD 구성시 실행이 되게 구성할 예정

### Hangma 서비스를 Docker Image로 구성
- Docker 컨테이너로 포트 4000에 실행된 Flask app에 호스트 운영체제(내컴퓨터)로 접근한다면?
    + 컴퓨터 안의 컴퓨터이기때문에 내 컴퓨터는(local)이 아니라 접속할 수 없다.
    + Docker 컨테이너 내부 프로세스가 오픈한 포트번호는 바깥 프로세스에서는 안 보임
    + 포트 맵핑을 사용해서 이를 해결할 수 있음
- Docker 컨테이너 내부 프로세스가 오픈한 포트번호를 외부로 노출해주는 것이 포트맵핑(포트 포워딩)
    + docker run 수행시 -p 옵션 사용
    + docker run -p 4000:4000 {Imaga Name}

### 커맨드라인 리뷰
```bash
cat > Dockerfile
FROM python:3.8-slim-buster             # baseimage
LABEL Maintainer="yyj024@naver.com"     # dockerimage에 대한 메타데이터(key-value)
WORKDIR /app                            
COPY app.py ./
COPY requirements.txt ./                
RUN pip3 install -r requirements.txt    # 이미지를 빌드할 떄 실행되는 명령어 
EXPOSE 4000에                           # 포트 맵핑을 할 때 참고하라는 정보
CMD ["python3","-m", "flask", "run", "--host=0.0.0.0", "--port=4000"] 
# docker 컨테이너가 실행될 때 실행되는 명령어
```

- docker build
```bash
docker build --platform=linux/amd64 -t yeojun024/hangman .
docker image ls
docker inspect yeojun024/hangman
docker run -p 4000:4000 yeojun024/hangman
docker run -p 4000:4000 -d yeojun024/hangman # 백그라운드에서 실행중 
docker ps
docker stop {container_id}
docker push yeojun024/hangman # docker hub에 repo를 만들지 않고 push(자동생성)
```

- docker image 삭제
```bash
docker ps -a --filter ancestor={image id}
docker rm {위에서 나온 container id}
docker rmi {삭제하려는 image id}
docker images #최종 확인
```
